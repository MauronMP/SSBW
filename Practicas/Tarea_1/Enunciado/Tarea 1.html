Tarea 1

Añadiendo test unitarios


Los test de código ayudan a mejorar la calidad del código, y son el primer paso para la Integración Continua. Además este tipo de código se presta para generarlo automáticamente con herramientas de IA.

En esta tarea añadiremos un par de pruebas unitarias al programa de la tarea anterior. Una de ellas comprobara el funcionamiento del servidor, verificando el código 200 OK, y otra comprobará un contenido concreto en la respuesta.

Vamos a usar las librerías MOCHA y Chai que proporcionan un entorno cómodo para los test, además podemos instalar la extensión Mocha Test Explorer.

Seguiremos los pasos de How to use Mocha/Chai to test Node.js apps, excepto que usaremos fetch en lugar de la librería request (desactualizada) que propone el articulo para hacer lass llamadas al servidor. Así prácticaremos el aspecto asíncrono de ES6 con las Promesas.

Es conviente añadir una orden para test en package.json: 

 ...
"scripts": {
	"dev": "npx nodemon app",
	"test": "mocha --reporter spec",
}
...

para que se pueda hacer 

> npm run test
